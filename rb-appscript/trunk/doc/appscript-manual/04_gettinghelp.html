4. Getting Help

Previous | Up | Next

Appscript provides two ways to get information about applications' scripting interfaces: the rbosadict shell program and introspective methods.

See the misc/rbosadict/README file for instructions on installing rbosadict. Note that py-appscript must be installed before rbosadict can be used.

Using rbosadict

rbosadict renders an application's terminology resource to readable HTML format. Its syntax is as follows:

rbosadict app-name html-file

    app-name -- name or full path of application

    html-file -- path to write HTML file to

Examples:

rbosadict /Applications/TextEdit.app ~/TextEdit.html

rbosadict Finder ~/Finder.html

Using introspective methods

Appscript Application and Reference objects provide several methods for obtaining information on the target application:

properties
Returns a list of all property names.
elements
Returns a list of all element names.
commands
Returns a list of all command names.
keywords
Returns a list of all class, enumerator, type and property names.
parameters(commandName)
Takes a command's name as string and returns the names of its parameters.

Like Ruby's standard methods method, all of these methods return a list of strings.

Examples

te = AS.app('TextEdit')

p te.commands
# ["activate", "close", "count", "delete", "duplicate", "exists", ...]

p te.parameters('make')
# ["at", "new", "with_data", "with_properties"]

Other sources of help

For advice and assistance with using rb-appscript:

  • Support forums for rb-appscript are available at http://rubyforge.org/forum/?group_id=2346
  • The sample folder contains example scripts demonstrating a range of common tasks.

For information on scripting specific applications:

  • Refer to any scripting documentation and example scripts supplied by the application developer.
  • The appscript entry at the pythonmac.org wiki links to pages that discuss scripting specific applications.
  • Look for third-party scripts that provide examples of use (though many of these scripts will be written in AppleScript rather than Ruby).
  • The MacScrpt mailing list at Dartmouth College discusses application scripting but extends to all Mac scripting languages.
  • MacScripter.net is the main hub for AppleScript-based application scripters, providing file archives, forums and links to other related sites.
  • Apple's website provides a developer section and mailing lists relating to application scripting and AppleScript.
