Implementation notes

aem 0.12.0+ creates its own Mach port each time an event is sent. This allows aem to send and receive events when used from non-main threads. (Earlier versions used the default Mach port, which meant that reply events could only be received on the main thread.) Note that this feature is not yet finalised and may be subject to further changes. In addition, it has not yet been extensively tested, so if any unexpected problems occur during use then please report them ASAP.

aem 0.10.0+ uses CarbonX.AE instead of Carbon.AE. Python scripts that refer to the AE extension directly should use the CarbonX version to ensure compatibility with aem.

Requires OS X 10.3+
