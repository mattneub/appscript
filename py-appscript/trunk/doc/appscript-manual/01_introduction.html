1. Introduction

What is appscript?

Python appscript (py-appscript) is a high-level, user-friendly Apple event bridge that allows you to control scriptable Mac OS X applications using ordinary Python scripts. Appscript makes Python an excellent alternative to Apple's own AppleScript language for automating your Mac.

For example, to get the value of the first paragraph of the topmost document in TextEdit:

app('TextEdit').documents['Read Me'].paragraphs[1].get()

This is equivalent to the AppleScript statement:

tell application "TextEdit"
    get paragraph 1 of document "Read Me"
end tell

Appscript includes:

  • a concise, elegant, object-oriented-like syntax for constructing references to application objects and sending application commands
  • a robust, flexible and mature architecture providing excellent compatibility with the wide range of scriptable OS X applications that are currently available
  • a powerful interactive help system for exploring application dictionaries and object models
  • multi-threading support
  • 32-bit support from Mac OS X 10.3.9 onwards, with 64-bit support on Mac OS X 10.5 and beyond.

Note that appscript's primary goal is to provide levels of functionality, reliability and application compatibility that equal to, or better than, AppleScript's own. Where Apple event behaviours differ from Python behaviours (e.g. Apple event elements are 1-indexed while Python lists are 0-indexed), appscript follows the Apple event approach to ensure the best possible compatibility with scriptable applications.
